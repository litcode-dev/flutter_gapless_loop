# Play-Once / Loop Toggle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `play({bool loop = true})` so callers can choose gapless looping (default) or one-shot playback that transitions to `PlayerState.stopped` at the end.

**Architecture:** The `loop` bool is sent via the method channel and stored as a native-side flag. Each platform reads the flag at schedule-time: iOS/macOS uses it to pick `AVAudioPlayerNodeBufferOptions`, Android uses it at the write-thread wrap point, Windows uses it to set `XAUDIO2_BUFFER.LoopCount`. All platforms add a `handlePlaybackComplete` / `PostPlaybackComplete` path that fires `stopped` when the flag is false and audio ends.

**Tech Stack:** Dart (Flutter method channel), Swift (AVAudioEngine, iOS 14+ / macOS 11+), Kotlin (AudioTrack MODE_STREAM), C++ (XAudio2 2.9)

**Spec:** `docs/superpowers/specs/2026-03-25-play-once-design.md`

    /**
     * Called from the write thread when one-shot playback reaches the natural end.
     *
     * Mirrors the teardown sequence of [stop()]:
     * 1. Captures [audioTrack] locally (not @Volatile; safe local snapshot).
     * 2. Pauses and flushes the AudioTrack to halt the hardware renderer.
     * 3. Resets [currentFrameAtomic] so [currentTime] returns to start.
     * 4. Dispatches [EngineState.Stopped] on the main thread via [engineScope].
     */
    private fun handlePlaybackComplete() {
        val track = audioTrack
        track?.pause()
        track?.flush()
        currentFrameAtomic.set(loopStartFrame.toLong())
        sessionManager.abandonAudioFocus()
        engineScope.launch { setState(EngineState.Stopped) }
    }
